# フロントエンドガイド

xyMusicUpdater のフロントエンドは、最大限のパフォーマンスと軽量な UI を実現するために、React 18、Vite、およびプレーンな CSS を使用してシングルページアプリケーション (SPA) として構築されています。

## 1. 分散型ポーリングとステート管理

Redux のようなグローバルなステート管理ライブラリに依存したり、すべてのエンドポイントを常にポーリングしたりする代わりに、フロントエンドはタブのコンテキストに基づいた「オンデマンド」の読み込み戦略を採用しています。

*   **コンテキストに基づく取得**: `/api/songs/` や `/api/playlist-map/` のような重いペイロードは、ユーザーがそれらを必要とするタブ（例：ライブラリ、タグ付け、エディター）に切り替えた場合に**のみ**厳密に要求されます。
*   **コールバックによる更新**: データを変更するすべての API 呼び出し（例：`triggerRescan`, `mergeCompilation`, `updateSong`）は、ターゲットを絞った `.then(refreshAll)` をトリガーし、古いデータを直ちに無効にしてビューを更新します。

## 2. Server-Sent Events (SSE)

`useSSE` フック (`frontend/src/hooks/useSSE.js`) は、バックエンドとの永続的でリアルタイムな接続を管理します。この接続は、システムの稼働状態を確認するための主要なハートビートメカニズムとしても機能します。

*   **回復力**: 接続が切断されたり、サーバーが再起動したりした場合、フックは再試行ポリシー（5 秒ごとに最大 10 回）を実行し、最終的に失敗した場合は強制的にログアウトさせます。
*   **イベントログ**: バックエンドは `info`、`warning`、`error` などのイベントをフロントエンドの `LiveLog` コンポーネントに直接ブロードキャストするため、ログファイルを手動でポーリングすることなく、バックグラウンドタスクの進行状況を直感的に確認できます。

## 3. UI の美学: Glassmorphism とテーマ設定

ユーザーインターフェースは、Material-UI や Tailwind CSS のような重いコンポーネントライブラリを避け、高度に最適化されたインラインスタイルと汎用的な CSS クラスに完全に依存しています。

*   **動的なテーマ設定**: ユーザーは設定パネルから `UI_THEME_COLOR` を設定できます。`App.jsx` コンポーネントはこれを CSS 変数 (`--accent`) として動的に注入し、すべてのボタン、境界線、アクティブな状態の色を瞬時に更新します。
*   **Glassmorphism (ガラスモーフィズム)**: `UI_DASHBOARD_BG` が有効な場合、UI はサイドバー、コンテンツパネル、フッターに `backdropFilter: blur(12px)` と半透明の RGBA 背景を適用します。これにより、ユーザーのカスタム背景画像が柔らかく透けて見えます。
*   **アニメーション**: `index.html` は、いくつかの中核となる CSS キーフレームアニメーション (`animate-slide`, `animate-slide-up`, `animate-fade`, `animate-bounce`) を定義しています。これらはグローバルに適用され、モダンで滑らかな触感のユーザーエクスペリエンスを保証します。
*   **無限マーキー**: `ScrollingText` コンポーネントは、DOM の `offsetWidth` に基づいてテキストのオーバーフローを自動的に検出します。オーバーフローしている場合、テキストを複製し、動的に計算された `animation-duration` を適用して、アプリ全体のすべての長いタイトルが 1 秒間に 40 ピクセルの一定の速度で滑らかにスクロールするようにします。
